# nusviz 3D 视图性能修复执行任务清单（React + Three.js + R3F）

> 背景：当前 `nusviz` 在播放、暂停、拖动时间轴、切换相机模式时存在明显的 CPU 开销偏高与帧时间抖动问题，瓶颈集中在每帧重复计算、相机投影链路、点云颜色计算和轨迹重建。  
> 开发者画像：执行人需具备 3 年以上前端工程经验，熟悉 React 18/19、TypeScript、R3F、Three.js BufferGeometry/Instancing、浏览器性能分析（Chrome Performance / Memory / React Profiler）与测试工程化（Vitest/Jest + Playwright）。

需求：

## 阶段1: 停止无效重算，改为“数据变化驱动更新”

- 梳理并改造以下组件更新触发条件：`PointCloud`、`ObjectBoxes`、`EgoTrajectory`、`EgoFutureTrajectory`、`ObjectFutureTrails`
- 禁止在 `isPlaying=false` 且 `currentFrameData` 未变化时执行几何重建与 buffer 重写
- 将更新条件统一收敛为：`currentFrameData`、`visibleLayers`、`futureTrajectorySeconds`、`cameraMode` 的实际变化
- 校验 `useFrame` 中仅保留必须的轻量同步逻辑，移除可迁移到状态变更时机的计算

阶段1完成后必须编写测试代码并验证：

- 单元测试：验证在暂停状态且 frame 未变化时，重建函数调用次数为 0
- 组件测试：验证 layer toggle 后仅对应图层触发一次更新

---

## 阶段2: 重构相机投影链路，消除重复矩阵计算与对象分配

- 在每帧维度为每个 camera 只计算一次 `world->camera` 变换矩阵，禁止在每个点投影时重复构造/求逆矩阵
- 重构 `buildCameraProjectedBoxes` 为批处理流程：先中心点可见性判定，再执行角点投影
- 移除投影链路中的高频临时对象创建，改为复用向量/矩阵与结果容器
- 统一 `CameraOverlayCanvas` 的数据转换为就地写入或复用结构，禁止每次渲染深层 `map` 复制
- 保证 6 路相机下对象数量增加时帧时间增长曲线可控

阶段2完成后必须编写测试代码并验证：

- 单元测试：投影结果数值正确性（给定固定输入，输出像素坐标与深度一致）
- 单元测试：矩阵计算调用次数验证（每 camera 每帧 1 次）
- 组件测试：相机 overlay 开启/关闭时渲染结果一致且无异常闪烁

---

## 阶段3: 压缩点云与轨迹计算成本，完成增量化改造

- 点云渲染改为“frame 切换时更新”，禁止静止帧重复计算 z-range 与颜色
- 轨迹链路改造为增量更新：`EgoTrajectory` 不得从 0 到当前帧全量重建
- `ObjectFutureTrails` 引入固定规则的轨迹点简化，保持视觉连续性并降低顶点/索引写入量
- `ObjectBoxes` 边线更新逻辑优化为最小必要写入，避免每帧全量逐顶点重复变换
- 确保优化后视觉语义不变：颜色、透明度、轨迹长度、选中高亮行为保持一致

阶段3完成后必须编写测试代码并验证：

- 单元测试：轨迹点简化前后路径误差在约束范围内（固定阈值）
- 单元测试：增量更新时 buffer 长度与 drawRange 变化符合预期
- 视觉回归测试：关键帧截图对比（点云、框体、轨迹）

---

## 阶段4: 强化数据加载与交互稳定性，封闭性能回归流程

- 为 `SceneDataManager.loadFrame` 增加 in-flight 请求去重，禁止同帧并发重复解析
- 对 timeline 高频拖动建立稳定加载顺序，避免竞态导致的重复解码和抖动
- 固化 S1~S6 性能测试矩阵为自动化脚本入口，统一结果输出格式（JSON）
- 在 CI 或本地脚本中加入性能回归门禁，未达阈值直接判定失败
- 输出最终性能报告：模块级收益、总收益、剩余瓶颈与后续修复清单

阶段4完成后必须编写测试代码并验证：

- 单元测试：同 frame 并发请求仅触发一次真实加载
- 集成测试：连续 seek 5 次无异常状态与无重复请求风暴
- 性能回归测试：完整跑 `S1~S6`，生成最终 before/after 报告并归档

---

## 全局执行约束（必须遵守）

- 每个阶段完成后，先补齐测试代码，再进入下一阶段
- 每次提交只允许包含一个阶段的改动与对应测试
- 性能结论必须基于同一机器、同一场景、同一测试矩阵的对比数据
- 未通过该阶段测试与性能回归，禁止进入下一阶段
